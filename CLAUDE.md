# プロジェクト概要

ボードゲーム「7 SHADES OF S:EVEN」リメイク（仮）のデジタル版アプリ。
姉妹プロジェクトとして、隣接フォルダ（親フォルダ直下）に「7 SHADES OF S:EVEN 戦績管理システム」
（プレイテスト結果の記録用アプリ、index.html）が別リポジトリとして存在するが、本プロジェクトとは無関係。

## ゴールと段階

- 最終ベンチマークは MTGA（Magic: The Gathering Arena）＝カード効果を完全に自動処理する形。
- ただしいきなりそこは目指さず、まずはユドンリウムコネクト（ユドコネ）のような
  「アナログコンポーネントを画面に移植しただけ」の形を目指す。カード効果はテキスト表示のみで、
  処理はプレイヤーの自己申告に委ねる。
- ルールブックやカード効果は今後変更される可能性がある前提で設計する。

## 対戦形態（現状の合意事項）

- 最終的にはオンライン非同期対戦（同時接続ではなく、各プレイヤーが自分のタイミングで手番を進める形）を目指す。
- 実装は1人用（対戦相手なしのテストプレイ用）から始める。
- ただし「1人用→オンライン非同期」の移行で手戻りが出ないよう、最初から下記のアーキテクチャ方針を徹底する。

## アーキテクチャ方針

- ゲーム状態（盤面・手札・山札・フェーズなど）を1つのシリアライズ可能なオブジェクト（JSON）として持つ。
- 状態変更は全て「アクション」として表現し、`現在の状態 + アクション → 次の状態` の純粋な形にする
  （Reduxライクな設計）。1人用の段階ではアクションをローカルで即時適用するだけにしておき、
  オンライン非同期化する際は同じアクションをバックエンドに保存して同期する流れにそのまま乗せる。
- カード効果・カードデータはコードに埋め込まず、外部データ（JSON等）として持つ（変更に強くするため）。

## 技術スタック（現状の合意事項）

- 姉妹プロジェクト（戦績管理システム）に倣い、ビルド不要の単一HTML＋Tailwind CSS（CDN）＋Supabase
  （オンライン非同期対戦のバックエンド）という軽量構成を踏襲する。
- プラットフォームはデスクトップアプリを最終形としたいが、まずはブラウザで動くものを作り、
  将来的にTauri等でラップしてデスクトップアプリ化する想定。スマホとPCのクロスプレイも目標。

## 設計

- ゲーム状態データ構造: [docs/game-state-design.md](docs/game-state-design.md)
  - GameState / Board / Player / CardDefinition / CardInstance / Action の設計（v1、ユーザー合意済み）。
  - 乱数処理の同期方針: 結果をアクションのpayloadに確定値として含める（1人用⇄オンライン非同期で同じロジックを使うため）。

## Phase 1 実装状況

- 方針: CSS疑似3D（`perspective`+`rotateX`）で開始。3D化が必要になった場合は描画層（`src/main.js`の描画処理）だけを差し替える想定で、ゲームロジックとは分離して実装する。
- モバイル対応: PC（横長ブラウザ）優先。スマホ横向きは意識するが、実機での細かい調整はPhase1完了後に回す（現状、縦幅が狭い画面では上下のプレイヤーゾーンが見切れる）。
- ローカル確認用サーバー: ルートの `.claude/launch.json` に `7shades-digital` という設定名で追加済み（`npx serve 7shades-digital -l 3001`）。ルート直下の`static-serve`(3000番)は姉妹プロジェクト用なのでポートを分けている。
- 静的レイアウト（盤面7x7・4人分のロックエリア/手札ゾーン・共有山札等）を [index.html](index.html) / [src/style.css](src/style.css) / [src/main.js](src/main.js) にダミーデータで実装済み。まだゲーム状態（GameState）とは未接続。
- **注意（ハマりどころ）**: CSS疑似3Dの箱に `class="table"` を付けたところ、Tailwind CDNの組み込みユーティリティクラス`table`（`display:table`）と衝突し、`display:grid`が上書きされるバグが発生した。Tailwind使用時はカスタムクラス名がTailwindの既存ユーティリティ名（`table`, `hidden`, `flex`, `grid`, `block`, `container`等）と衝突しないよう注意する（今回は`game-table`に変更して回避）。
